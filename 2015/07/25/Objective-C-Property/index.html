Objective-C Property

Posted on 2015-07-25 | In Objective-C

Property Introduction

Jie Meng

Property
前世今生

  1. @property
    属性（@property ）用于自动生成setter和getter，即accessor，以及默认生成实例变量。
    1.1 手动生成accessor方法：
    没有property之前，类声明及实现实例变量的存取方法如下：
    People.h
    #import <Foundation/Foundation.h>
    @interface People: NSObject
    { NSString _name; //instance variable
    }
    (void) setName: (NSString     )peopleName; //setter
    (NSString )name; //getter
    #end
    People.m
    #import “People.h”
    @implementation People
    (void) setName: (NSString     )peopleName{
    _name = peopleName;
    }
    (NSString *)getName{
    return _name;
    }
    如代码所示，创建存取器需要在头文件中声明，并在.m文件中实现setter和getter。而property可以自动创建存取器。

1.2 自动生成accessor方法：
@property (strong, nonatomic, copy) NSString *name;
头文件中这一句代码即实现了setter和getter，使用时即可用方法setName，亦可用点方法 .name；还自动生成了实例变量_name（即自动生成代码@synthesize name = _name;）。（代码括号中的内容下节说明）
Note：
如果使用了@property后仍然手动生成setter或getter方法，则property不会再自动生成setter或getter；
如果同时手动生成setter和getter，则property不会自动生成相应带下划线的实例变量；此时若需要实例变量，需在.m文件中使用@synthesize声明。

  1. property attributes（特性）
    上节@property后的括号内使用了三个特性。实际上，property有多个特性：atomic, nonatomic, strong, weak, retain, readwrite, readonly, copy, assign, setter=, getter=； 其中atomic，strong，readwrite，assign是默认特性。
    可将上述关键字按用途分为三类：原子性、存取器控制、内存管理。
    2.1 原子性：
    原子性是property在线程环境下的行为。如果有不止一个线程，setter和getter可能同时被调用，会引发数据错误。
    atomic保证了setter和getter执行安全，不会引发冲突；但是线程安全并非仅此一方面，所以保证存取结果不冲突不代表线程安全。iOS的多线程并非是一个对象上设置多个线程，比如model 和 UI会进行线程间通信。
    nonatomic是非线程安全的，但是在实际应用中，基本都是nonatomic，因为atomic在系统中实际并不完全安全，并且占用大量资源，处理时间长。 并且，atomic下，setter和getter必须同时被property自动生成或者同时被用户定义，不然会告警“Writable atomic property cannot pair a synthesised getter/setter with a user defined setter/getter”. 而nonatomic可以混用两种。

2.2 存取器控制：

使用setter=和getter=特性可以更改默认存取器的名字，例如：
@property (getter=isRunning) BOOL running;
此时，使用getter不再是[instanceName running]了，而是[instanceName isRunning]；但注意，此时点方法仍使用默认名称.running。
readwrite是默认值，默认setter和getter都存在；而readonly表示只有getter，没有setter。

2.3 内存管理：
strong，weak，copy，retain，unsafe_unretained和assign都是内存管理相关的关键字。
由于现在普遍使用ARC(自动引用计数)，所以着重讲下ARC引用的关键字：strong、weak、copy代表了不同的ownership关系。
strong
strong是强引用，且是默认特性，拥有对象的所有权。一旦某个property是strong，则其直到没有任何对象引用到这个property，即自动计数为0，其才会被释放；且是自动释放；无需手动添加release代码。
weak
weak不拥有对象的所有权，也不会对传入对象进行引用计数加1的操作。基本只在可能引起retain circle的情况下（会造成内存泄露）才使用weak，否则都使用strong。
只要自动计数为0，weak对象就会被释放，且置为nil。
copy
copy与strong类似，但它对传入对象的副本拥有所有权，而非对象本身。
通常情况下，当某个property是指向其他对象的指针，且该对象的类有可修改的子类（例如NSString/NSMutableString或NSArray/NSMutableArray）时，应将该属性的内存管理特性设为copy。

手动引用计数MRR：
以前的retain等同于ARC下的strong。在ARC下不再使用。
而unsafe_unretained类似于weak，只是其声明的对象在释放后并不自动置为nil，可能造成悬垂指针。
assign在iOS5之前用于对象和weak作用等同，但之后不再使用。现在仅用于值类型，如int、float、double和NSInteger，CGFloat等表示单纯的复制。还包括不存在所有权关系的对象。
